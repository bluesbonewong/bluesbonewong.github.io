发表于 2018-05-09 | 分类于 面试题

¶1. HTTP 状态码知道哪些？

  • 101,200,202,204,301,302,303,304,307,403,404

¶101

101表示按照其上的头信息变为一个不同的协议

¶202/204表示什么

202服务器已接受请求，但尚未处理 204服务器已处理请求，但没有返回任何内容

¶403/404表示什么

403被拒绝 404资源未找到——一般是请求路径出了问题

¶303和307的区别

303和307状态码是对302的细化，有着相同的功能：都是临时重定向，但是303明确表示客户端应当采用get方法获取资源，307则是重新发起post请求

¶304表示什么

304是指文件未改变，使用http缓存

¶2. 301 和 302 的区别是什么？

  • 301：永久重定向
  • 302：临时重定向

¶3. HTTP 缓存怎么做？

  1. 没有缓存：浏览器会反复向Server请求资源，浪费带宽
  2. 只有缓存：如果Server的资源变了，浏览器无法获取最新资源
  3. 缓存+更新Expires：第一次请求某个资源时，Server -> 浏览器，附带一个额外信息 —— 过期时间， 如使用Expires，浏览器把这个时间储存在本地，等到下次请求相同资源时， 将浏览器时间与Expires的时间进行比对，查看是否过期，过期则从服务端拉取最新资源， 未过期则使用本地缓存 缺点：不可靠，比如本地时间出现问题，比Server端的时间提前了一年，则还是会反复请求
  4. 升级版缓存+更新Catch-Control：第一次请求资源，服务器返回完整文件与附带信息Catch-Control:max-age=300; 在时间内300s内的请求，使用缓存，超过这个时间则再次发送网络请求 可选值： 1.Public：响应可被任何中间节点缓存 2.Private：中间节点不允许缓存 3.no-catch：不使用catch-control做前置验证，而是直接使用Etag或Last-modified字段来控制缓存 4.no-store：真正的不缓存任何东西，也不使用Etag 5.max-age 缺点：以时间控制，万一东西其实一直没有更新，那请求还是浪费的
  5. 终极版缓存+更新Catch-Control + Etag字段：第一次请求，服务器给请求的资源返回一个Etag字段，它是对该资源的编码, Catch-Control过期之后，再次请求相同资源，会对Etag字段进行比对在请求头里的if-none-match中， 相同返回304，不相同则对该资源重新请求 缺点：如果有多个服务器，请求资源万一是随机发到所有服务器上的，那么Etag都不一样，还是会重新请求
  6. 究极版缓存+更新Catch-Control + Last-Modified/if-Modified-since：第一次请求，服务器给请求的资源返回一个Last-Modified字段, 记录该资源在Server端的最后修改时间，再次请求该资源会在请求头中带上if-Modified-since， 记录了上一次请求返回的Last-Modified，Server接收到了请求后，会对它们进行对比，若Server端的Last-Modified更新， 则响应新资源，否则返回304

¶7. GET 和 POST 的区别是什么？

  • 参数：GET的参数写在请求头中，POST写在请求体中
  • 参数长度限制GET的长度限制在1024个字节，POST的长度限制在4~10Mb
  • 功能区别：GET是获取资源
  • 安全：在http协议下，都不安全，但是相对而言，POST比GET安全，GET的请求信息直接暴露在URL里，post放在了请求体内

¶8. 怎么跨域？JSONP 是什么？CORS 是什么？postMessage 是什么？

  • JSONP是通过 script 标签加载数据的方式去获取数据当做 JS 代码来执行
  • CORS设置响应头中的Access-Control-Allow-Origin

¶9. TCP 和 UDP 的区别是什么

简答：TCP可靠、面向连接、相对 UDP 较慢； UDP 不可靠，不面向连接、相对 TCP 较快。

¶10. TCP 的三次握手指的是什么

  1. 客户端：我要连接你了，可以吗
  2. 服务端：嗯，我准备好了，连接我吧
  3. 客户端：那我连接你咯。
  4. 开始后面步骤

¶11. AJAX的readyState几种状态

0 UNSENT 代理被创建，但尚未调用 open() 方法。 1 OPENED open() 方法已经被调用。 2 HEADERS_RECEIVED send() 方法已经被调用，并且头部和状态已经可获得。 3 LOADING 下载中； responseText 属性已经包含部分数据。 4 DONE 下载操作已完成。

¶13. 说一下dns

  • domain name system，域名系统
  • 它作为将域名和IP地址相互映射的一个分布式数据库，能够使人更方便地访问互联网

¶14. 谈一谈性能优化

¶第一部分：加载的优化

  • 网络越快越好：把网站静态资源放在CDN上，服务器网速要好
  • 东西体积越少越好：服务端将资源压缩，前端将资源【特别是图片】压缩
  • 东西数量越少越好：减少请求，比如说把icon变成字体
  • 服务器处理的越快越好：
  • 能重复利用的最好重复利用：比如http的缓存，相同的css、js文件预先加载

¶第二部分：体验的优化

  • 只给用户暂时需要的东西
  • 别让用户感觉卡顿
  • 动画不要卡

¶15. 谈一下https

¶1.保证数据传输的安全——https

http的请求都不安全

¶2.三种加密方式

  • 对称加密AES：加密和解密使用同一个秘钥。特点：速度快
  • 非对称加密RSA：公钥加密的必须用私钥解密，私钥加密的必须用公钥加密。特点：速度慢，cpu开销大
  • Hash加密MD5/SHA1：无法反向解密，一般用来验证数据未被修改

¶3.https的原理 —— CA链逐级认证

  • 当一个网站要使用 https 时先需在一些国际认证的 CA 机构填写网站信息申请证书，而这些 CA机构往往还有上层 CA，最终有一个根 CA。
  • 一般来说浏览器都会内置根 CA 和一些顶级 CA 的证书，但需要验证的时候会通过 CA 链逐级验证。

¶16. 谈一谈XSS攻击

  1. 输入框使用了innerHTML，恶意人员输入script标签并且带上了恶意代码
  2. 避免：
  • 不要使用innerHTML
  • 对用户的输入进行转义

¶17. 谈一谈CSRF攻击

跨站请求伪造Cross Site Request Forgery
