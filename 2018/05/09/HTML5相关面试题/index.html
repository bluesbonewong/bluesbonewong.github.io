¶1. Doctype作用？标准模式与兼容模式各有什么区别?

  • Doctype用来规范浏览器行为，HTML5不需要写<!DOCTYPE HTML>，只需要写<!DOCTYPE>就行，但是HTML4.0需要，否则会导致文档以兼容模式呈现
  • 标准模式【严格模式】的排版和js运行机制都是以该浏览器支持的最高标准运行；兼容模式【混杂模式】以宽松的向后兼容的方式显示

¶3. 设置行内元素的padding和margin有效果吗？

¶7. 浏览器本地存储的常见方式有哪些？

¶7.1 Cookie

  1. cookie是存储在浏览器上的一小段数据，用来记录某些当页面关闭或者刷新后仍然需要记录的信息，比如用户名密码等
  2. 浏览器访问指定域名是必须带上 Cookie 作为 Request Header，所以Cookie不宜太大，否则会浪费带宽，一般4k
  3. Cookie的主要作用就是用来识别用户身份【判断该用户之前是否已经访问过】，而非储存数据

¶7.2 LocalStorage

  1. LocalStorage是window下的一个API，设置方式为window.localStorage.setItem('搜索记录',searchHistory)，获取方式为getItem
  2. 大小一般在5m，一般存储用户的例如购物车信息等
  3. 只能存储字符串，会将你储存的数据格式都转化为字符串
  4. 浏览器即使关闭，LocalStorage依然存在

¶7.3 SessionStorage

与LocalStorage类似，唯一不同的是其生命周期不一样 ——浏览器被关闭后，自动清空，注意浏览器刷新不属于浏览器被关闭

¶8. 如何设置cookie，以及如何设置cookie过期(具体代码)

1.document.cookie设置和查看Cookie document.cookie = "max-age:1000;" 设置cookie过期时间，或者是设置expires来控制过期时间 2.由后端设置，一般会在response header里有一个[set-cookie]来设置Cookie

¶9. Session是什么

  • Session 是服务器端的内存（数据），用来标识用户
  • Session 一般通过在 Cookie 里记录 SessionID 实现
  • SessionID 一般是随机数
