¶3.JS 中typeof能得到哪些类型

  1. 七个数据类型中，除了typeof null === 'object'，其它都正常
  2. 此外还会返回一个function

¶4. JS 变量按照储存方式区分为哪些类型，并描述其特点

  • 原始类型（primitive type）和复杂类型（complex type）
  • 原始类型是最基本的数据类型，不能再细分了，一般为String、Number、Boolean
  • 复杂类型往往是多个原始类型的值的合成，可以看作是一个存放各种值的容器，可再细分为狭义的对象（object）、数组（array）、函数（function）
  • undefined和null一般被认为是两个特殊的值

¶5. undefined和null的异同

  • 二者都表示空
  • undefined是undefined类型，null是null类型
  • 它在 JavaScript 内部有一些使用场景（比如声明原型链的终结 Foo.prototype = null），但是大多数情况下都可以使用 undefined 来代替

¶6.谈谈你对JSON的理解

  1. 在JS看来，JSON就是一个内置对象，其提供了两个API——JSON.parse(),JSON.stringnify()
  2. 另一方面，JSON也是一种数据格式

¶7. 如何准确判断一个变量是数组类型

  1. 用Object.prototype.toString.call([])
  2. 用Array.isArray([])

¶8.如何理解作用域

  1. let出来之前，js没有块级作用域，只有函数和全局作用域
  2. 变量会沿着当前作用域->父级作用域->全局作用域去查找它的声明
  3. 讲一讲闭包相关的东西

¶11 这段代码里的 this 是什么？严格模式下又是什么？

  • 全局调用 —— window【undefined】
  • 嵌套函数 —— window【undefined】 下面严格模式是一样的
  • 方法调用 —— 指向其父级对象
  • 数组【对象的一种】 —— 指向其父级对象
  • setTimeout —— window
  • 事件 —— 触发事件的DOM节点
  • 使用构造函数的this —— 实例

¶12. 什么是立即执行函数？使用立即执行函数的目的是什么？

  • 声明之后立刻执行
  • 目的是为了制造一个独立的作用域，防止变量污染

¶13. async/await 语法了解吗？目的是什么？

  • await 操作符用于等待一个Promise 对象。它只能在异步函数 async function 中使用
  • 目的是将异步代码写成同步代码

¶20. call 、apply 和 bind 的用法和区别是什么？

共同作用： 调用函数（bind不调用函数，而是返回新函数） 改变该函数this值 操作参数 区别： call接受一个个的参数，apply是一个数组 call/apply返回其调用函数的返回值 bind返回一个新的函数，调用这个新函数其实就是把原函数call/apply一下，并指定你传递的this
